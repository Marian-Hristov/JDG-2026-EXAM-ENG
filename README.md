## üìÅ Repo Structure

The mandates to be completed are located in this folder. Each mandate contains:

- a `README.md` with the context and general instructions for the mandate.
- `QX` subfolders (one question per subfolder), each with its own `README.md` and the provided input files / templates.

#### Global Structure:

- `2-SomniaData/` (databases / modeling and SQL queries)

Read carefully each `README.md` of the question (`QX`) you are working on: it specifies the expected deliverables, the output format, as well as the provided input files.

## üìÑ Submission

You have two options to submit your finalized work:

1. **Via GitHub**: Fork this repo, complete the exercises in your fork, then create a Pull Request before the end of the exam. Ensure that your fork is private and shared only with your team members during the exam, otherwise other teams could have access to your solutions.
2. **Via ZIP + Google Form**: Compress the folder **of one mandate at a time** in `.zip` format and send it via the submission Google Form. Clearly identify your team as well as the mandate number in the file name (e.g., `TeamPoly_Mandate7.zip`) and in the Google Form.
